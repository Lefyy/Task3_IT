# Задача 3 — Моделирование полёта квадрокоптера

Данный проект реализует **вариант 3** задания по моделированию процессов: симуляцию полёта квадрокоптера с вероятностными событиями, многопоточностью и событийной архитектурой.

## Постановка задачи

В рамках задания необходимо:

- смоделировать полёт квадрокоптера;
- реализовать минимум 3 сущности:
  - `Квадрокоптер`;
  - `Оператор`;
  - `Механик` (через интерфейс);
- использовать:
  - **делегаты/события**;
  - **многопоточность**;
  - **рефлексию** (где необходимо);
- обеспечить динамическое поведение модели на форме (движение объектов);
- поддержать добавление **нескольких моделей** одновременно.

## Что реализовано в проекте

- Оператор включает пульт управления для конкретного дрона.
- Квадрокоптер начинает движение в пределах области симуляции.
- Во время полёта может случиться вероятностное событие: **отключение GPS**.
- При аварии запускается аварийная посадка.
- После посадки механик выполняет ремонт и восстанавливает GPS.
- После ремонта дрон снова может быть запущен оператором.
- На форме можно добавить несколько квадрокоптеров, и каждый работает как отдельная модель.

## Как в проекте выполнены требования задания

### 1) Делегаты и события

- `Operator.ControllerTurnedOn` — событие включения пульта.
- `Quadcopter` публикует события:
  - изменения позиции;
  - отключения GPS;
  - начала аварийной посадки;
  - завершения посадки.
- `Mechanic` использует делегат `RepairMessageHandler` и событие публикации сообщений о ремонте.

### 2) Многопоточность

- Полёт дрона и аварийная посадка выполняются асинхронно (`Task.Run`, `Task.Delay`, `CancellationToken`).
- Для потокобезопасного доступа к состоянию дрона используется синхронизация (`lock`).

### 3) Рефлексия

- Экземпляр механика создаётся через reflection:
  - поиск конкретной реализации `IMechanic` в сборке;
  - создание через `Activator.CreateInstance(...)`.

## Краткая инструкция по запуску

### Предварительные требования

- Установленный **.NET SDK 10** (или совместимый preview SDK для `net10.0`).

### Шаги запуска

1. Перейти в каталог проекта:

   ```bash
   cd /workspace/Task3_IT
   ```

2. Восстановить зависимости:

   ```bash
   dotnet restore
   ```

3. Запустить приложение:

   ```bash
   dotnet run
   ```
